# 代码审查清单

在提交代码前，确保通过以下检查项。

## 类型安全

- [ ] 所有组件都有 TypeScript 类型
- [ ] 没有使用 `any` 类型
- [ ] Props 接口定义完整
- [ ] API 响应有类型定义
- [ ] 使用 `import type` 导入类型

## UI 组件（shadcn/ui）

- [ ] 使用了 shadcn/ui 组件而非自定义样式
- [ ] 组件从 `@/components/ui` 导入
- [ ] 正确使用 Button 变体（default/secondary/destructive/outline/ghost/link）
- [ ] 表单使用了适当的输入组件（Input、Textarea、Select 等）
- [ ] 使用 Lucide React 图标而非其他图标库

## 样式（Tailwind CSS）

- [ ] Tailwind 类名按逻辑分组排列（布局→间距→外观→交互）
- [ ] 使用 CSS 变量（如 `bg-card`、`text-primary`）而非硬编码颜色
- [ ] 支持深色模式（使用 `dark:` 前缀或 CSS 变量自动切换）
- [ ] 响应式设计完整（移动端优先，使用 `md:`、`lg:` 等断点）
- [ ] 使用 `cn()` 工具函数合并类名

## 状态管理

- [ ] 状态放在最近的共同父组件
- [ ] 异步操作有 loading/error 状态
- [ ] useEffect 有正确的依赖数组
- [ ] useEffect 有清理函数（如需要）

## 国际化（i18next）

- [ ] 所有用户可见文本使用 `t()` 函数
- [ ] 语言包键名符合命名规范（camelCase，点分隔层级）
- [ ] 中英文语言包同步更新
- [ ] 使用 `Trans` 组件处理带标签的翻译文本

## API 调用（TanStack Query）

- [ ] 使用 Query Hooks 而非直接 fetch
- [ ] Query Key 命名规范（按资源分组）
- [ ] 处理 loading/error/empty 状态
- [ ] 错误消息友好展示给用户（使用 toast）
- [ ] 突变操作后调用 `invalidateQueries` 刷新缓存

## 可访问性

- [ ] 图片有 alt 属性
- [ ] 图标按钮有 aria-label
- [ ] 表单输入有关联的 label
- [ ] 使用语义化 HTML 标签

## 性能

- [ ] 列表渲染有稳定的 key
- [ ] 复杂计算使用 useMemo
- [ ] 传递给子组件的函数使用 useCallback
- [ ] 没有在渲染中创建对象/数组

## 代码质量

- [ ] 没有未使用的导入
- [ ] 没有 console.log（调试代码）
- [ ] 组件职责单一
- [ ] 复杂逻辑提取到 Hook
- [ ] 文件不超过 300 行

## 命名

- [ ] 组件使用 PascalCase
- [ ] 函数使用 camelCase
- [ ] 布尔值使用 is/has/can 前缀
- [ ] 事件处理函数使用 handle 前缀

## 测试准备

- [ ] 组件可以独立测试
- [ ] 副作用可以被 mock
- [ ] 没有硬编码的测试障碍

## 快速自检命令

```bash
# pnpm 包管理
pnpm install

# TypeScript 类型检查
pnpm run typecheck

# ESLint 代码检查
pnpm run lint

# 格式化检查
pnpm run format:check

# 构建检查
pnpm run build
```
