# AI 前端开发与测试规范（React 19 + TypeScript 5.8 + Tailwind 4）

> 本规范用于指导 **AI 与开发人员** 在 YouMind 前端项目中进行一致、可验证、可自动化的开发与测试工作。
>
> **技术栈**: React 19 + TypeScript 5.8 + Tailwind 4 + pnpm + Vite

---

## 一、总体原则

1. **先设计接口，再写实现，最后写测试**（AI 也必须遵守）
2. **新增功能 = 代码 + 测试**，禁止只提交实现不提交测试
3. 测试必须：

   * 可自动执行
   * 不依赖真实后端 API
   * 可在本地和 CI 中运行
4. 所有测试必须通过以下命令：

```bash
pnpm test
```

---

## 二、测试分层策略

```
Component Test       → 测组件渲染和交互
Hook Test            → 测自定义 Hooks 逻辑
Service/API Test     → 测 API 调用和数据处理
Integration Test     → 测页面级集成
```

### 明确不做

* ❌ 不做端到端 E2E 测试（由 Playwright 负责）
* ❌ 不测试 React 框架行为
* ❌ 不测试 Tailwind 样式

* 不允许把测试集中到单独的 test 包中

## 四、组件开发与测试规范

### 4.1 组件设计规范

* 使用 **函数组件 + Hooks**
* 优先使用 **shadcn/ui** 组件库
* 样式使用 **Tailwind CSS**

```typescript
// 组件示例
import { Button } from '@/components/ui/button';

interface UserCardProps {
  name: string;
  email: string;
  onEdit?: () => void;
}

export function UserCard({ name, email, onEdit }: UserCardProps) {
  return (
    <div className="p-4 bg-white rounded-lg shadow">
      <h3 className="text-lg font-semibold">{name}</h3>
      <p className="text-gray-600">{email}</p>
      <Button onClick={onEdit} variant="outline" className="mt-2">
        编辑
      </Button>
    </div>
  );
}
```

### 4.2 组件测试规范

* 使用 **Vitest + React Testing Library**
* 覆盖：

  * 正常渲染
  * Props 传递
  * 用户交互
  * 错误边界

```typescript
import { render, screen, fireEvent } from '@testing-library/react';
import { describe, it, expect, vi } from 'vitest';
import { UserCard } from './index';

describe('UserCard', () => {
  it('should render user name and email', () => {
    render(<UserCard name="Tom" email="tom@example.com" />);
    
    expect(screen.getByText('Tom')).toBeInTheDocument();
    expect(screen.getByText('tom@example.com')).toBeInTheDocument();
  });

  it('should call onEdit when button clicked', () => {
    const handleEdit = vi.fn();
    render(<UserCard name="Tom" email="tom@example.com" onEdit={handleEdit} />);
    
    fireEvent.click(screen.getByText('编辑'));
    expect(handleEdit).toHaveBeenCalledTimes(1);
  });
});
```

### 强制要求

> **只要新增或修改组件，必须同步新增或修改测试**

## 五、Hook 开发与测试规范

### 5.1 Hook 设计规范

* 单一职责
* 返回清晰的状态和操作方法

```typescript
import { useState, useCallback } from 'react';

export function useCounter(initial = 0) {
  const [count, setCount] = useState(initial);
  
  const increment = useCallback(() => setCount(c => c + 1), []);
  const decrement = useCallback(() => setCount(c => c - 1), []);
  const reset = useCallback(() => setCount(initial), [initial]);
  
  return { count, increment, decrement, reset };
}
```

### 5.2 Hook 测试规范

```typescript
import { renderHook, act } from '@testing-library/react';
import { describe, it, expect } from 'vitest';
import { useCounter } from './useCounter';

describe('useCounter', () => {
  it('should initialize with default value', () => {
    const { result } = renderHook(() => useCounter());
    expect(result.current.count).toBe(0);
  });

  it('should increment count', () => {
    const { result } = renderHook(() => useCounter());
    
    act(() => {
      result.current.increment();
    });
    
    expect(result.current.count).toBe(1);
  });
});
```

## 六、API Service 开发与测试规范

### 6.1 Service 设计规范

* 使用统一的 API 客户端
* 定义清晰的请求/响应类型

```typescript
// services/userApi.ts
import { apiClient } from '@/lib/api-client';

export interface User {
  id: number;
  name: string;
  email: string;
}

export interface CreateUserRequest {
  name: string;
  email: string;
}

export const userApi = {
  getUsers: () => apiClient.get<User[]>('/api/users'),
  getUser: (id: number) => apiClient.get<User>(`/api/users/${id}`),
  createUser: (data: CreateUserRequest) => apiClient.post<User>('/api/users', data),
  updateUser: (id: number, data: Partial<User>) => 
    apiClient.put<User>(`/api/users/${id}`, data),
  deleteUser: (id: number) => apiClient.delete(`/api/users/${id}`),
};
```

### 6.2 Service 测试规范

* 使用 **MSW** 拦截 HTTP 请求
* 测试成功和失败场景

```typescript
import { describe, it, expect, beforeAll, afterAll, afterEach } from 'vitest';
import { setupServer } from 'msw/node';
import { http, HttpResponse } from 'msw';
import { userApi } from './userApi';

const server = setupServer();

beforeAll(() => server.listen());
afterEach(() => server.resetHandlers());
afterAll(() => server.close());

describe('userApi', () => {
  it('should fetch users', async () => {
    const mockUsers = [
      { id: 1, name: 'Tom', email: 'tom@example.com' },
    ];
    
    server.use(
      http.get('/api/users', () => {
        return HttpResponse.json(mockUsers);
      })
    );
    
    const users = await userApi.getUsers();
    expect(users).toEqual(mockUsers);
  });

  it('should handle error', async () => {
    server.use(
      http.get('/api/users', () => {
        return new HttpResponse(null, { status: 500 });
      })
    );
    
    await expect(userApi.getUsers()).rejects.toThrow();
  });
});
```

## 七、AI 编码强制约定（必须遵守）

AI 在生成或修改前端代码时，必须遵循以下规则：

1. 每一个新增组件或 Hook，**必须同时生成对应测试**
2. 不允许：

   * 只生成实现代码
   * 生成无法通过的测试
3. 组件测试必须覆盖：
   * 渲染测试
   * Props 测试
   * 交互测试
   * 错误边界测试
4. 所有代码提交前，测试必须通过：

```bash
pnpm test
```

## 八、CI 执行规范（最低要求）

```yaml
- name: Run frontend tests
  run: |
    pnpm install
    pnpm test --run
```

### 合并门禁

* 测试失败 → 禁止合并
* 新功能无测试 → 禁止合并

## 九、验收标准

* 新增功能具备对应测试
* 可一键执行全部测试
* 测试不依赖外部系统
* CI 自动校验通过
